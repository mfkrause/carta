## Findings considered and rejected (do not re-audit)

- **Grid table cells rendered twice (measure pass + final render; `plain.rs:410/445`,
  `rst.rs:960/995`)** — real, but the two renders use different widths (measure at unbounded
  width, emit at fitted width), so it's measure-then-layout by design, not pure redundancy.
  Deferred until plan 001 shows table rendering matters in practice; a caching fix has MED
  risk for an unproven win.
- **`auto_grid_widths` quadratic loop (`grid.rs:513`)** — already reduced from cubic
  (commit 08f7944); quadratic in column count is negligible at realistic table widths.
- **`Extensions::iter` scans all 11 variants (`extensions.rs:129`)** — cold path, trivial N.
- **HTML tree `locate()` linear rescans (`tree.rs:199`)** — constant small work per document.
- **`unique_id` allocates per collision candidate (`convert.rs:384`)** — collisions are rare
  and bounded by heading count.
- **Cross-phase link-label normalization caching** — the single-pass rewrite in plan 004
  captures most of the win; a block→inline cache adds plumbing for little remaining gain.
- **CLI output buffering** — vetted fine: the CLI writes the whole converted output with one
  `write_all`; no per-line syscall problem exists.
- **JSON codec intermediate trees** — vetted fine: `codec.rs` uses direct
  `serde_json::from_slice`/`to_string`, no intermediate `Value`.
